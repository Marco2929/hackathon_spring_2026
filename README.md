# Agents YouTube Shorts Pipeline

This repository automates a full product-to-video workflow for YouTube Shorts:
(Branch max_setup includes the maximum version)
1. Pull Amazon product links from a local queue.
2. Scrape and structure product details and images.
3. Analyze product images with vision.
4. Write a short-form script with scene and video prompts.
5. Generate scene images with OpenRouter FLUX.2 Klein 4B.
6. Generate narration audio with Edge TTS.
7. Animate scene with local video generation model Wan 2.2 5b via ComfyUI.
8. Fuse clips + audio into one final MP4.
9. Upload the final video to YouTube.

The core orchestration is implemented in the CrewAI project under `youtube_crew/`.

## Prerequisites

- Python 3.10 to 3.13
- uv package manager
- ffmpeg and ffprobe available in PATH
- Chromium installed for Playwright
- ComfyUI server reachable over network
- OpenRouter API key
- Google Cloud OAuth credentials for YouTube upload

## Setup

### 1) Install dependencies

```bash
cd youtube_crew
uv sync
uv run playwright install chromium
```

### 2) Environment variables

Create `youtube_crew/.env` (or export in shell):

```env
# Required
OPENROUTER_API_KEY=your_openrouter_key

# Optional OpenRouter image overrides
OPENROUTER_IMAGE_MODEL=black-forest-labs/flux.2-klein-4b
OPENROUTER_CHAT_URL=https://openrouter.ai/api/v1/chat/completions

# Optional ComfyUI overrides
COMFYUI_HOST=192.168.178.128
COMFYUI_PROMPT_URL=http://192.168.178.128:8188/prompt
COMFYUI_WORKFLOW_PATH=/absolute/path/to/YouTube_gen_smol.json
COMFYUI_DOWNLOAD_DIR=/absolute/path/to/youtube_crew/output/comfyui
COMFYUI_SAVE_NODE_ID=58

# Optional YouTube uploader overrides
YOUTUBE_CLIENT_SECRETS=/absolute/path/to/client_secret_xxx.json
YOUTUBE_TOKEN_FILE=/absolute/path/to/youtube_token.json
```

### 3) Prepare Amazon link database (if needed)

From repository root:

```bash
python amazon_links_db/setup_db.py
```

## Run the Pipeline

From `youtube_crew/`:

```bash
uv run python -m youtube_crew.main
```

Equivalent project script:

```bash
uv run run_crew
```

On each run, `output/` is reset and regenerated by `youtube_crew/src/youtube_crew/main.py`.

## Active Agent/Task Flow

The current sequential flow in `youtube_crew/src/youtube_crew/crew.py` is:

1. `manage_link_queue`
2. `extract_product_data`
3. `analyze_product_task`
4. `create_shorts_script_task`
5. `generate_scene_images_task`
6. `create_tts_audio_task`
7. `generate_video_clips_task`
8. `fuse_video_task`
9. `upload_youtube_video_task`

### Important tools in use

- `amazon_content_scrape_tool`: Product data extraction
- `multi_image_description_tool`: Multi-image visual analysis
- `openrouter_scene_image_generator`: Scene image generation (OpenRouter FLUX)
- `edge_tts_generate_audio`: Narration generation
- `ComfyUI Video Generator`: Scene-to-video rendering
- `video_fusion_tool`: Clip concat + audio mux
- `youtube_data_api_uploader`: Upload to YouTube

## Notes on Prompt Formats

- `generate_scene_images_task` expects scene descriptions and image paths as semicolon-separated lists for `openrouter_scene_image_generator`.
- `generate_video_clips_task` follows the same semicolon-separated format for scene/video generation.

## Troubleshooting

- If image generation fails: verify `OPENROUTER_API_KEY` and model access.
- If video generation fails: verify ComfyUI host/port, workflow node IDs, and save node output.
- If upload fails: verify OAuth client file, token file, and YouTube API enablement in Google Cloud.
- If fusion fails: ensure `ffmpeg` and `ffprobe` are installed and in PATH.
